# 产销平衡系统-三块核心能力产品流程图（文本版）

## 0. 文档目的
- 固化“预警跟进处理闭环、诊断与决策闭环、模拟建议闭环”三块能力
- 作为后续建表、接口、前端新页签开发的统一追溯基线
- 当前版本按轻量可落地原则设计，先跑通闭环再逐步扩展

## 1) 预警跟进处理闭环（执行闭环）

### 1.1 流程图（文本）
```
数据采集
  -> 规则触发预警
  -> 生成预警卡(alert_id)
  -> 分级路由(P1/P2/P3)
  -> 责任人领取
  -> 关联主建议(recommendation_id)
  -> 人工确认(采纳/驳回)
  -> 执行追踪(execution_id)
  -> 完成/异常关闭
  -> 复盘归档
```

### 1.2 轻量状态机
```
预警状态: 新建 -> 处理中 -> 已关闭/已忽略
执行状态: 待确认 -> 执行中 -> 已完成/已驳回
```

### 1.3 最小字段
- alert_id
- alert_level
- alert_type
- affected_org_name
- sku_name
- gap_qty_ton
- fulfillment_rate_pred
- recommendation_ref_id
- owner_user
- action_deadline

### 1.4 预警来源定义（新增）
- 自动触发：由数据监控规则引擎触发，不依赖人工录入
- 手工触发：由业务人员提交“事件工单”触发，不采用直接录表方式
- 预警配置应支持阈值、连续天数、异常波动三类规则
- 预警展示支持非结构化内容：标题+描述+上下文信息，可直接进行指派与跟进

### 1.5 自动触发规则样例（新增）
- 库存安全预警：库存低于安全库存/高于上限库存
- 连续异常预警：连续2-3天库存过高或过低
- 价格异常预警：单价偏离近7日均值超过阈值
- 订单异常预警：异常取消率、延期率、集中改单等

### 1.6 监测总览口径（新增）
- 总触发数：历史累计触发预警总量
- 待处理：未关闭且未处理完成
- 超时：超过完成时间仍未关闭/未处理完成
- 已关闭：已完成闭环并关闭

## 2) 诊断 -> 调整建议 -> 人工决策闭环（决策闭环）

### 2.1 流程图（文本）
```
接收预警(alert_id)
  -> 自动诊断(需求/供给/物流/经营)
  -> 输出诊断结论(主因+次因+影响量化)
  -> 生成方案集(主建议1 + 备选A/B)
  -> 方案评分(履约优先、成本次之、库存健康)
  -> 人工决策(接受主建议/接受备选/微调后接受/驳回)
  -> 形成已确认建议
  -> 推送执行闭环
```

### 2.2 诊断四域定义
- 需求域：订单结构、区域动销、客户优先级、指定提货组织约束
- 供给域：可用库存、产线能力、品种结构、可调拨空间
- 物流域：车次能力、线路时效、跨组织成本
- 经营域：价格、单吨贡献、毛利稳定性

### 2.3 建议生成原则
- 先硬约束后优化：先满足履约与提货组织约束，再做利润与成本优化
- 固定输出结构：主建议（唯一）+ 备选A + 备选B
- 每个建议都要有动作、效果、风险三段
- 建议来源采用混合引擎：大模型解释 + 运筹算法求解 + 规则兜底

### 2.5 建议与采纳交互定义（新增）
- 建议类型：自动建议（算法/模型）与人工建议（专家补充）
- 处理动作：采纳、驳回、转人工、补充说明、处理完成反馈
- 必留痕字段：处理人、处理时间、处理结果、结果说明
- 辅助建议形态：针对单条预警给出“可采纳建议卡”，支持一键采纳并进入执行
- 系统事件流支持“模型处理建议”一键触发，为负责人提供默认处置方案

### 2.4 人工决策动作
- 接受主建议
- 接受备选建议
- 微调参数后接受（数量/时点/优先级）
- 驳回并填写理由

## 3) 模拟建议闭环（利润最大化）

### 3.1 目标
- 在履约与安全约束下实现利润最大化

### 3.2 流程图（文本）
```
选择模拟场景
  -> 输入冲击参数(价格/成本/产能异常)
  -> 约束校验(履约下限、库存安全线、提货组织约束、产能上限)
  -> 引擎计算(主建议+备选A/B)
  -> 输出利润、履约率、库存天数、风险对比
  -> 人工选定方案
  -> 下发执行
  -> 回写实际结果(预测vs实际)
```

### 3.3 场景模板
- 场景A：价格上涨/下调 X%
- 场景B：成本上涨 X%（原燃料、电、运费）
- 场景C：产能异常（某基地/产线下降 X%）

### 3.4 关键输出
- 主建议动作清单（调价、调拨、产能结构调整）
- 预测利润变化
- 履约率变化
- 库存天数变化
- 风险提示（客户影响、执行难度）

### 3.5 模拟变动因子（新增）
- 产能（基地/产线）
- 售价（按区域）
- 需求（区域/客户）
- 成本（原燃料、电、人工）
- 运输成本（线路/车型/时效）

### 3.6 运筹学引擎建议（新增）
- 目标函数：利润最大化（兼顾履约率与库存安全）
- 约束条件：产能、库存、需求满足、运输能力、客户提货组织约束
- 输出方案：主建议1套 + 备选2套，并给出边际收益与执行风险

## 4) 新页签规划（不动原页，原内容做参考）

### 4.1 页签建议
- 页签1：产销舆情中心（预警舆情+辅助建议+跟进处置同页）
- 页签2：模拟建议（利润最大化，单独页签）

### 4.2 页面关系
```
预警跟进(执行结果入口)
   ^            |
   |            v
诊断与决策 ----> 已确认建议
   |
   v
模拟建议(策略沙盘) -> 选定后可回流诊断与决策
```

## 5) 建表与开工顺序（建议）

### 5.1 第一批（先开做）
- alert_events（预警主表）
- recommendations（建议主表，含主建议/备选）
- executions_light（执行轻量跟踪表）

### 5.2 第二批（稳定后再上）
- diagnosis_snapshots（诊断快照）
- simulation_runs（模拟运行记录）
- simulation_actions（模拟动作明细）

### 5.3 开工检查点
- 检查点A：三张核心表+最小接口能闭环
- 检查点B：前端三页签能串起来
- 检查点C：至少一条真实业务链路跑通并可复盘
