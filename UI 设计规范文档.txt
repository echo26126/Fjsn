 UI 设计规范文档
一、设计总则
1. 设计目标
确保数据可视化大屏在深色科技驾驶舱与浅色清新商务看板两种风格下，视觉统一、交互友好、信息高效传递。
兼顾大屏监控的实时性与日常分析的易用性，提升用户决策效率。
2. 适用范围
企业运营中心、智慧城市、工业互联网、销售管理、CRM 等数据可视化场景。
覆盖 PC 端大屏、桌面端看板等多终端适配。
二、视觉规范
1. 色彩系统
（1）深色科技驾驶舱风格

高亮主色	#00D4FF / #00A8FF	青蓝、湖蓝，用于核心数据、图表主色
强调色	#FF7D00 / #7B61FF	橙色（警示）、紫色（强调），用于异常数据
辅助色	#36CFC9 / #FFB800	浅绿、浅黄，用于次要图表、数据标签
文字色	#FFFFFF（主） / #8A919F（次）	白 / 浅灰，保证深色背景下的可读性
（2）浅色清新商务看板风格
类型	色值（HEX）	说明
背景主色	#F5F7FA / #FFFFFF	浅白 / 浅灰，营造清爽商务感

强调色	#FF7D00 / #F53F3F	橙色（增长）、红色（下降 / 异常）
辅助色	#86909C / #E5E6EB	浅灰（次要文字）、浅灰（边框 / 分割线）
文字色	#1D2129（主） / #4E5969（次）	深灰 / 中灰，保证浅色背景下的可读性
2. 字体规范
字体类型	推荐字体	应用场景	字号范围
主标题	Inter / 思源黑体 Bold	大屏标题、模块标题	24px - 36px
副标题	Inter / 思源黑体 Medium	图表标题、数据卡片标题	16px - 20px
正文 / 数据	Inter / 思源黑体 Regular	明细数据、表格内容	12px - 16px
辅助说明	Inter / 思源黑体 Light	同比说明、单位标注	10px - 12px
注：大屏场景下，字号需按分辨率等比放大，确保 3 米外清晰可读。
3. 布局与组件
（1）深色科技驾驶舱风格
布局：强分区网格布局，模块间距 8px - 16px，模块边框为 1px 高亮渐变线（#00D4FF → #00A8FF），带轻微发光效果（box-shadow: 0 0 8px rgba (0, 212, 255, 0.3)）。
组件：
数字卡片：深色背景 + 高亮数字，支持动态翻牌效果。
图表：折线图带粒子流动效果，柱状图带渐变填充，饼图带扫描动画。
交互：hover 时模块边框高亮，数据 tooltip 为半透明深色背景。
（2）浅色清新商务看板风格
布局：卡片式网格布局，卡片圆角 8px - 12px，间距 24px，卡片阴影为 0 2px 8px rgba (0,0,0,0.08)。
组件：
数字卡片：浅白背景 + 圆角，同比标签为圆角胶囊（如同比 + 8.6%）。
图表：折线图为柔和渐变填充，柱状图为半透明主色，环形图为多色分段。
交互：hover 时卡片轻微上浮（transform: translateY (-2px)），数据 tooltip 为浅白背景 + 细边框。

4. 图层交互规范
hover 交互：鼠标悬浮区域时，显示该区域名称、核心数据（如销售额、订单量），tooltip 样式与整体 UI 一致：
深色风格：tooltip 背景 #1E293B（80% 透明），文字 #FFFFFF，边框 1px #00D4FF；
浅色风格：tooltip 背景 #FFFFFF，文字 #1D2129，边框 1px #E5E6EB，阴影 0 2px 4px rgba (0,0,0,0.08)。
点击交互：点击区域可下钻（如从大区到战区），地图缩放比例适配数据粒度，缩放时有 0.3s 平滑过渡动画。
动画效果：数据更新时，区域色块渐变过渡（transition: all 0.5s ease），避免突兀闪烁；标记点有脉冲呼吸动画（深色：rgba (0,212,255,0.6) → 透明；浅色：rgba (22,119,255,0.6) → 透明）。
四、交互规范
1. 核心交互
数据刷新：支持自动刷新（5s/10s/30s）与手动刷新，刷新时数据卡片有翻牌 / 渐变动画。
维度切换：支持按品牌 / 单品、大区 / 战区、时间维度切换，切换时图表平滑过渡（transition: all 0.3s ease）。
下钻分析：点击图表数据点、地图区域，可跳转至明细页面或弹窗展示详情，弹窗样式与整体风格统一。
2. 反馈机制
操作反馈：hover、点击时的颜色 / 阴影变化，确保用户感知交互状态；
加载反馈：数据加载时显示骨架屏或加载动画（深色：旋转的青蓝渐变圆环；浅色：品牌蓝进度条），避免空白等待。
