# 生产环境创建｜数据平台任务书

## 1. 目标

- 在生产环境完成数据分析平台的部署与数据通路打通，支撑产销看板、查询报表与优化模型的数据底座。
- 确保 Doris 分析库稳定运行，ETL 同步链路可靠，Excel 上传通道可用且有审核机制。

## 2. 范围

- 在范围内：
  - **Apache Doris 部署与调优**
    - Doris FE/BE 集群部署（单机或多节点，按实际规模）
    - 维度表与事实表创建（对齐 PRD 数据模型）
    - 物化视图与汇总表配置
    - 查询性能调优与资源隔离
  - **MySQL 业务库部署**
    - 用户/权限/配置/日志/上传记录等业务表
    - 与 Doris 的数据同步关系
  - **ETL 同步服务**
    - 中间库 → Doris 的 ETL 任务配置
    - 增量/全量同步策略
    - 同步监控与异常告警
  - **Excel 上传服务**
    - 上传、校验、解析、暂存、审核入库全链路
    - 模板管理与下载
    - 单位换算（自动转为万吨）
  - **数据初始化**
    - 历史数据导入与清洗
    - 主数据初始化（基地、区域、产品）
    - 成本/物流参数表初始化
  - 数据备份与恢复策略
- 不在范围内：
  - ERP/OA 源系统改造
  - 数据治理与主数据管理平台建设（仅做本项目必需的初始化）

## 4. 部署架构

### 4.1 Doris 集群部署

```
┌──────────────────────────────────────────────┐
│              Apache Doris 集群               │
│                                              │
│  ┌─────────┐    ┌─────────┐    ┌─────────┐  │
│  │  FE-1   │    │  BE-1   │    │  BE-2   │  │
│  │ (Leader)│    │ (数据)  │    │ (数据)  │  │
│  └─────────┘    └─────────┘    └─────────┘  │
│       │              │              │        │
│       └──────────────┼──────────────┘        │
│                      │                       │
│              Query / Load                    │
└──────────────────────────────────────────────┘
```

- 最小部署：1 FE + 1 BE（开发/演示）
- 推荐部署：1 FE + 2~3 BE（生产）
- 资源建议：
  - FE：4C8G（元数据管理、查询规划）
  - BE：8C16G + SSD 200G+（数据存储与计算）

### 4.2 ETL 同步架构

```
ERP/OA ──► 中间库 ──►┐
                     ├─► ETL 服务 ──► Doris（分析）
Excel 上传 ─► 暂存表 ┘       └──► MySQL（业务）
```

- ETL 工具选型：DataX / Flink CDC / 自研脚本（按实际选择）
- 同步频率：
  - 事实数据：日批次 T+1（可配置更高频）
  - 维度数据：变更触发或日同步
  - Excel 上传：实时（审核通过即入库）

## 7. 验收标准

- Doris 可用：
  - 集群健康，核心查询 <5 秒（月度聚合场景）
  - 数据模型完整，维度表/事实表/视图与 PRD 一致
- ETL 可靠：
  - 日批次同步正常运行 ≥3 天无异常
  - 同步失败可告警，可手工重跑
- Excel 上传可用：
  - 模板下载、上传、校验、审核、入库全链路跑通
  - 单位换算正确，异常数据拦截有效
- 数据完整：
  - 主数据初始化完毕，抽样对账通过
  - 口径统一（万吨）且与前端展示一致
