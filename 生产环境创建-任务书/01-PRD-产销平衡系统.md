# 产销平衡系统｜PRD（福建能化-福建水泥集团）

## 1. 背景与目标

- 客户：福建能化下属单位福建水泥集团
- 建设目标：围绕"产、销、库"形成统一视图与决策建议，辅助集团与各基地/区域进行产销协同，**以利润最大化为核心目标**，运用运筹学优化方法解决供需错配、库存波动与区域调配问题。
- 核心价值主张：
  - **看得清**：统一数据口径，实现全省产销一张图
  - **问得到**：AI 智能问数，自然语言查询业务数据
  - **算得准**：运筹学优化模型，产销平衡利润最大化
  - **管得住**：预警提示+可解释建议，辅助科学决策

## 2. 范围与阶段

- 本期范围（4块内容）：
  1. 产销情况看板（驾驶舱/指挥中心）
  2. 按时间维度查询生产计划（年度、月度）与销售（出库）情况
  3. 数据接入管理（接口对接 + Excel 上传）
  4. 产销平衡优化与建议（**核心重点**）
- MVP分阶段交付：
  - **第一阶段——"看得见 + 问得到"**：
    - 数据底座搭建（MySQL + Doris）
    - 数据接入（ERP/OA 中间库 + Excel 手工上传）
    - 驾驶舱看板 + 侧边栏 AI 问数
    - 时间维度查询 + 侧边栏 AI 问数
    - DeepSeek 本地部署 + Text-to-SQL 调优
  - **第二阶段——"算得准 + 管得住"**：
    - 产销平衡规则引擎（阈值预警 + 供需缺口分摊）
    - **运筹学优化模型（利润最大化线性规划/MILP）**
    - 预警与建议生成（可解释性）
    - 建议方案的模拟推演与对比
  - **第三阶段（后续）——"全局优化，闭环调度"**：
    - 直接生成可执行方案（引入 DeepSeek V3 大模型能力 + 优化算法联合求解）
    - 多目标优化（利润最大化 + 客户满意度 + 库存健康度）
    - 闭环调度与方案自动下发

## 3. 用户与使用场景

- 集团管理层：快速掌握全省产销态势与风险（缺口、库存高、价格波动），查看**利润最大化方案**的全局视图。
- 生产计划员：查看年度/月度计划与实际偏差，调整基地排产与发运节奏，接收**优化排产建议**。
- 销售管理：查看区域销量与价格结构，跟踪目标与出库达成，接收**区域调配与定价建议**。
- 基地调度/仓储：关注本基地产量、库存、出库、可用库存预警，接收**库存周转优化建议**。
- 数据管理员：负责 Excel 数据上传、审核确认、模板管理。

## 5. 功能需求

### 5.1 产销情况（看板/驾驶舱）

- 展示形态：
  - 驾驶舱大屏 / 看板页面（标题：**产销平衡指挥中心**）
  - 福建地图可视化展示
  - 支持切换"生产情况/销售情况"
  - **左下角 AI 问数助手悬浮窗**（全站可用）
- 生产情况视图（按生产基地）：
  - 地图点位/标注：基地位置、名称
  - **产品品类维度**：水泥、熟料、骨料（分品类展示与筛选）
  - 指标（至少包含）：
    - 产量：当日/当月累计/当年累计【按品类拆分：水泥/熟料/骨料】
    - 库存：当前库存、可用库存（如区分锁定/在途）【按品类拆分】
    - 销售出库：当日/当月累计【按品类拆分】
  - **计量单位**：统一使用 **万吨** 作为计量单位
  - 交互：
    - 时间筛选：日/周/月/自定义区间
    - 品类筛选：水泥/熟料/骨料（支持全部/单选/多选）
    - 基地点击下钻：趋势、结构、明细表
    - 排名：基地产量、库存、销售出库排行
- 销售情况视图（按销售区域）：
  - 地图分区/区域层级展示：销售区域边界或区域聚合标注
  - **产品维度**：按 **型号** 分类，区分 **袋装/散装** 包装形态
  - 指标（量与价）：
    - 销售出库量：当日/当月累计/当年累计【按型号×袋散拆分】
    - 价格：均价、价格区间（如能取到）【按型号×袋散拆分】
    - 金额：销售额（如能取到）
    - **利润贡献**：各区域利润贡献占比（如可获取成本数据）
  - **计量单位**：统一使用 **万吨** 作为计量单位
  - 交互：
    - 区域点击下钻：客户/产品型号/袋散结构、趋势、明细表
    - 型号与袋散筛选：支持按型号和包装形态筛选
    - 排名：区域销售出库量、均价、销售额、**利润贡献**排行

### 4.2 时间维度查询（生产、库存、销售）

> 查询模块分为**生产查询**、**库存查询**、**销售查询**三大板块，支持按时间维度（年度/月度/日）进行多维度数据检索与对比分析。

- 通用查询维度：
  - 时间：年度、月度（必要时补充日维度用于对账）
  - 产品品类：水泥、熟料、骨料（支持全选/单选/多选）
- **计量单位**：统一使用 **万吨**
- 输出：
  - 图表 + 表格
  - **导出（Excel）为必选功能**
  - **左下角 AI 问数助手悬浮窗**，支持自然语言查询当前页面数据

#### 4.2.1 生产查询（按生产基地）

- 组织维度：**按生产基地**（支持单选/多选/全部基地对比）
- 产品维度：按品类——水泥、熟料、骨料
- 查询内容：
  - 年度计划、月度计划产量【按品类拆分】
  - 实际产量【按品类拆分】
  - 对比：计划 vs 实际产量、偏差（绝对值/百分比）
  - 产能利用率：实际产量 / 产能上限
  - 可选：开停窑/检修标记【如有】
- 展示形态：
  - 基地产量排名（柱状图/表格）
  - 计划 vs 实际偏差对比图
  - 产量趋势图（按月/按日）
  - 各基地产能利用率雷达图/仪表盘

#### 4.2.2 库存查询（按生产基地）

- 组织维度：**按生产基地**（支持单选/多选/全部基地对比）
- 产品维度：按品类——水泥、熟料、骨料
- 查询内容：
  - 期初库存、期末库存【按品类拆分】
  - 可用库存（= 总库存 - 冻结量 - 在途未卸量）【按品类拆分】
  - 在途库存【如有】
  - 库存变动：入库量（产量）vs 出库量（销售发运）
  - 库容占比：当前库存 / 仓容上限（百分比）
  - 安全库存达标情况：当前库存 vs 安全库存下限
  - 库存周转天数（可选）：当前库存 / 日均出库量
- 展示形态：
  - 基地库存水位图（液位计/进度条，标注安全线与仓容上限）
  - 库存趋势图（按月/按日，含安全库存线与仓容线）
  - 各基地库存排名与库容占比
  - 库存结构分析（品类占比饼图）
  - 入库 vs 出库对比趋势图

#### 4.2.3 销售查询（按生产基地 + 销售区域）

- 组织维度：**按生产基地 + 销售区域**（支持交叉筛选）
  - 可按"生产基地"查看该基地发往各区域的销售情况
  - 可按"销售区域"查看该区域来自各基地的供应情况
  - 支持基地×区域矩阵视图
- 产品维度：按**型号**，区分**袋装/散装**
- 查询内容：
  - 年度/月度销售出库量【按型号×袋散拆分】
  - 对比：目标 vs 实际（如存在销售目标），偏差（绝对值/百分比）
  - 量价联动：销售出库量、均价、金额（可用则展示）
  - 利润贡献（如可获取成本数据）：各基地×区域的利润贡献
  - 客户结构（如有）：重点客户销售占比
- 展示形态：
  - 基地→区域供货矩阵热力图（行=基地，列=区域，值=出库量/金额）
  - 区域销售排名（柱状图/表格）
  - 量价联动趋势图（双轴：出库量+均价走势）
  - 目标 vs 实际达成率对比图
  - 袋散结构占比分析

### 4.3 数据接入管理

> 数据接入支持两种方式并行：**系统接口对接** + **Excel 手工上传**，确保数据来源的灵活性与完整性。

#### 4.3.1 系统接口对接

- 对接路径：ERP/OA 系统 → 中间库 → ETL → Doris（分析库）
- 对接方式：
  - 定时数据抽取（中间库模式，客户侧 ERP/OA 推送或系统主动拉取）
  - 支持 RESTful API 接口对接（如客户系统支持）
  - 支持数据库直连抽取（中间库读取，只读，不写入客户源系统）
- 数据流向：

  ```
  ERP/OA ──► 中间库 ──► ETL ──► Doris（分析事实表/维度表）
                           └──► MySQL（业务配置/用户/权限/日志）
  ```

- 同步频率：T+1（日批次同步），看板类数据支持更高频更新
- 接口规范：
  - 提供标准化数据接口文档（字段映射、类型、频率）
  - 支持增量/全量同步模式
  - 同步任务监控与异常告警

#### 4.3.2 Excel 手工上传

- 适用场景：
  - ERP/OA 尚未对接的补充数据
  - 计划类数据（年度计划、月度计划、销售目标等）
  - 临时数据修正/补录
  - 成本/价格等敏感数据的定期更新
- 功能设计：

  | 功能 | 说明 |
  |------|------|
  | 模板下载 | 提供标准化 Excel 模板（生产日报、月度计划、销售目标、成本表等） |
  | 格式校验 | 列名/数据类型/必填项/枚举值/计量单位自动校验 |
  | 数据预览 | 上传后预览合法行 + 异常行高亮提示 |
  | 单位换算 | 上传数据自动转换为系统标准单位（万吨） |
  | 审核确认 | 上传人确认 或 管理员审批后入库 |
  | 重复检测 | 同日期+同基地数据覆盖策略（提示/覆盖/追加，可配置） |
  | 上传记录 | 保留每次上传的版本/时间/操作人，支持回溯 |
  | 数据回退 | 支持按批次撤回已上传数据 |

- 数据流向：

  ```
  用户上传 Excel ──► 上传服务（校验+解析）──► 暂存表 ──► 审核通过 ──► Doris
  ```

### 5.4 智能问数（AI 问答）

> AI 问数助手以 **左下角悬浮窗** 形式嵌入全站所有页面，用户可随时呼出进行自然语言查询，窗口支持隐藏/展开、拖拽移动、自由缩放。详细交互设计见 4.0.5。

#### 4.4.1 处理链路

```
用户自然语言提问 → 意图识别（DeepSeek）
   ├─ 数据查询 → Text-to-SQL → Doris 执行 → 自动可视化 → 自然语言总结
   ├─ 预警分析 → 规则引擎 → 自然语言总结
   ├─ 归因解释 → 推理链 → 自然语言总结
   └─ 闲聊/越界 → 安全兜底 → 提示回归业务
```

#### 4.4.2 Text-to-SQL 设计

| 环节 | 做什么 | 关键实现 |
|------|--------|---------| 
| Schema 注入 | 将 Doris 表结构+字段注释注入 Prompt | 维护"语义字典"：业务术语→字段映射 |
| Few-shot | 提供典型问答样例 | 维护 20-50 个高频问答样例 |
| SQL 生成 | DeepSeek 生成 Doris SQL | 限定 SELECT only，禁止 DDL/DML |
| 安全校验 | SQL 白名单校验 | 表名/函数白名单 + 行数限制 + 超时 |
| 权限注入 | 按用户角色追加 WHERE 条件 | 基地用户只查自己基地数据 |
| 执行 & 格式化 | Doris 执行并选择图表 | 根据维度数/度量数自动选择图表类型 |

#### 4.4.3 语义字典（示例）

| 业务术语 | 映射目标 | 备注 |
|---------|---------|------|
| 产量 | `fact_production.actual_qty` | 单位：万吨 |
| 计划产量 | `fact_production.plan_qty` | 单位：万吨 |
| 库存 | `fact_inventory.end_qty` | 默认取期末，单位：万吨 |
| 销售/出库 | `fact_outbound.qty` | 销售=出库口径，单位：万吨 |
| 均价 | `fact_outbound.avg_price` | 含税/不含税需确认，单位：元/吨 |
| 产能利用率 | `actual_qty / capacity` | 需关联维度表，百分比 |
| 完成率 | `actual_qty / plan_qty * 100` | 计算字段，百分比 |
| 利润 | `fact_outbound.profit` | 收入-成本，单位：万元 |

  - 可自由缩放大小（边缘拖拽，最小 320x400px）

### 5.5 产销平衡优化与建议（核心重点）

> 本模块是系统的核心价值所在。目标是在满足各类约束的前提下，**求解利润最大化的产销平衡方案**，运用运筹学方法进行科学决策。

#### 4.5.1 业务目标

- **核心目标**：在满足产能约束、库存约束、需求约束、物流约束等条件下，**最大化集团整体利润**。
- 辅助目标：
  - 最小化库存波动风险（避免爆仓停机/缺货断供）
  - 最大化客户需求满足率
  - 最小化物流运输总成本
  - 均衡各基地产能利用率

#### 4.5.2 运筹学优化模型

##### （1）决策变量

| 变量 | 含义 | 单位 |
|------|------|------|
| $x_{i,j,k}$ | 基地 $i$ 向区域 $j$ 供应产品 $k$ 的量 | 万吨 |
| $p_{i,k}$ | 基地 $i$ 产品 $k$ 的排产量 | 万吨 |
| $s_{i,k}$ | 基地 $i$ 产品 $k$ 的期末库存 | 万吨 |

##### （2）目标函数

$$
\max Z = \sum_{i,j,k} \big( \text{售价}_{j,k} \cdot x_{i,j,k} \big) - \sum_{i,k} \big( \text{生产成本}_{i,k} \cdot p_{i,k} \big) - \sum_{i,j,k} \big( \text{物流成本}_{i,j} \cdot x_{i,j,k} \big) - \sum_{i,k} \big( \text{库存成本}_{i,k} \cdot s_{i,k} \big)
$$

即：**利润 = 销售收入 - 生产成本 - 物流成本 - 库存持有成本**

##### （3）约束条件

| 约束类型 | 约束表达式（简化描述） | 说明 |
|---------|----------------------|------|
| **产能约束** | $p_{i,k} \leq \text{Capacity}_{i,k}$ | 各基地各品类排产量不超过产能上限 |
| **需求约束** | $\sum_{i} x_{i,j,k} \leq D_{j,k}$（或 $= D_{j,k}$） | 区域需求满足（可设为弹性约束） |
| **最低供应约束** | $\sum_{i} x_{i,j,k} \geq D^{min}_{j,k}$ | 重点客户/区域最低保供量 |
| **库存平衡** | $s_{i,k} = s^{0}_{i,k} + p_{i,k} - \sum_{j} x_{i,j,k}$ | 期末库存 = 期初 + 生产 - 发运 |
| **库存上下限** | $S^{min}_{i,k} \leq s_{i,k} \leq S^{max}_{i,k}$ | 安全库存 ≤ 库存 ≤ 仓容上限 |
| **运输能力** | $\sum_{k} x_{i,j,k} \leq T_{i,j}$ | 基地到区域的运输能力限制 |
| **检修约束** | 特定时段 $p_{i,k} = 0$ | 检修期间产能为零 |
| **非负约束** | $x_{i,j,k}, p_{i,k}, s_{i,k} \geq 0$ | 所有变量非负 |

##### （4）求解方法分阶段演进

| 阶段 | 方法 | 适用场景 | 工具/技术 |
|------|------|---------|-----------|
| **一期（规则引擎）** | 阈值预警 + 启发式分摊 | 快速上线、简单场景 | 自研规则引擎 |
| **二期（线性规划）** | LP / MILP 求解 | 标准产销平衡优化 | PuLP / OR-Tools / Gurobi |
| **三期（智能优化）** | DeepSeek + 优化求解器联合 | 复杂多目标+不确定性 | 大模型 + Meta-heuristic |

##### （5）一期规则引擎详细逻辑

在正式引入运筹学求解器之前，一期先通过规则引擎实现基础优化：

- **供需缺口计算**：
  - 各区域需求预测 vs 可供应量（产能 + 库存 - 安全库存）
  - 缺口 = 需求 - 可供应量，缺口 > 0 触发预警
- **就近优先分配**：
  - 按运输距离/成本排序，优先从最近基地调拨
  - 考虑运输能力约束
- **利润贡献排序**：
  - 按产品×区域的利润贡献度排序
  - 高利润区域/产品优先满足
- **库存均衡**：
  - 库存超上限的基地优先减产或增加外发
  - 库存低于安全线的基地优先补货或减少外发

##### （6）二期优化模型实施路径

1. **数据准备**：
   - 生产成本矩阵（基地×品类）
   - 物流成本矩阵（基地×区域）
   - 售价矩阵（区域×品类×型号）
   - 产能上限、仓容上限、安全库存
   - 需求预测（历史数据 + 趋势外推）
2. **模型构建**：
   - 使用 Python 建模（PuLP / OR-Tools）
   - 支持多周期滚动优化（月度/周度）
   - 支持场景模拟（What-if 分析）
3. **结果输出**：
   - 最优排产计划（各基地各品类排产量）
   - 最优调拨方案（基地→区域的供货分配）
   - 预计利润与敏感性分析
   - 可解释的优化报告（为何如此分配、瓶颈在哪）

#### 4.5.3 预警与建议输出

- 输入（来自数据层）：
  - 生产侧：各基地计划产量、实际产量、产能上限/约束、库存、**生产成本**
  - 销售侧：各区域需求/出库、价格、重点客户订单【如有】、**销售收入**
  - 物流/约束：运输能力/距离/成本、区域供货优先级【如有】
- 输出（建议最小集）：
  - **预警**：
    - 缺货风险预警（库存低于安全库存 + 需求缺口为正）
    - 库存高位预警（库存接近仓容上限）
    - 供需偏差预警（实际 vs 计划偏差超阈值）
    - **利润异常预警**（区域利润率低于阈值、量价背离）
  - **建议**：
    - 基地排产调整建议（增产/减产/结构调整）
    - **最优区域供货分配方案**（基地→区域的建议量，以利润最大化为目标）
    - 预计库存变化与达标情况（目标库存区间）
    - **利润影响评估**（调整方案的预计利润变化）
  - **解释**：使用了哪些数据与规则、触发阈值、**优化模型求解状态与约束分析**
  - **模拟推演**：
    - What-if 分析（如改变某参数的影响）
    - 多方案对比（保守/均衡/激进方案的利润与风险对比）
